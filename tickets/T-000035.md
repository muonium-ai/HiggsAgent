## Goal
Define reusable benchmark workloads and manifests that express comparable ticket shapes without live repository mutation.

## Acceptance Criteria
- [ ] Define a benchmark workload manifest shape that captures comparable ticket-shape inputs such as work type, platform, complexity, execution-target assumptions, and tool profile.
- [ ] Add an initial fixture corpus with representative benchmark workloads that are safe to share and do not depend on arbitrary live repository state.

- [ ] Expose the workload corpus in a repository location and shape that later harness and reporting work can consume deterministically.
