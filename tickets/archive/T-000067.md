## Goal
Send an explicit HiggsAgent app name and user agent in OpenRouter requests so provider telemetry no longer reports the default Python urllib identity.

## Acceptance Criteria
- [x] OpenRouter requests sent by HiggsAgent include an explicit application identifier instead of leaving `app_id` null in provider-side metadata.
- [x] OpenRouter requests sent by HiggsAgent identify the client as `HiggsAgent` instead of the default `Python-urllib/3.13` user agent.
- [x] The hosted OpenRouter transport has focused automated coverage proving the outgoing request carries the configured HiggsAgent identity metadata.
- [x] Failure-free execution paths continue to work without changing the existing OpenRouter response parsing or runtime contracts.

## Notes
- Observed OpenRouter generation metadata on 2026-03-08 showed `app_id: null` and `user_agent: "Python-urllib/3.13"` for a HiggsAgent-driven request.
- The motivating provider payload also reported `model: "openai/gpt-5.4-20260305"`, `provider_name: "OpenAI"`, `api_type: "completions"`, and `streamed: true`, so the fix should apply at the shared OpenRouter HTTP transport layer rather than one narrow call site.
- The shared OpenRouter HTTP transport now sends `HTTP-Referer: https://github.com/muonium-ai/HiggsAgent`, `X-OpenRouter-Title: HiggsAgent`, and `User-Agent: HiggsAgent` by default.
