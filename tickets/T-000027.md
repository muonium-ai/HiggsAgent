## Goal
Implement a normalized provider abstraction that allows hosted and local executors to share execution result shapes, observability outputs, and guardrail-aware interfaces.

## Acceptance Criteria
- [x] Introduce a shared execution abstraction that can represent hosted and local executors without changing downstream validation and observability shapes.
- [x] Preserve normalized execution result fields for output text, usage data, retry counts, events, and attempt summaries.
- [x] Keep provider-specific request and response handling behind hosted and local adapters.
- [x] Document or encode how partial local usage metrics are represented when exact billing data is unavailable.
