## Goal
Build the command-line interface for running built-in, file-based, and seeded Game of Life simulations.

## Acceptance Criteria
- [ ] Add a CLI entrypoint that can run a named built-in pattern, a file-loaded pattern, or a seeded/generated board.
- [ ] Support generation count and relevant board-size options without introducing ambiguous flags.
- [ ] Print stable terminal-friendly output that works for evaluator demos and test assertions.
- [ ] Keep CLI orchestration separate from the domain logic implemented in earlier tickets.

## Notes
- This is the main demo surface users will interact with.
- Animation is optional if the final output mode stays easy to demonstrate.

## Agent Assignment
- Suggested owner: cli
- Suggested branch: feature/game-of-life-cli

## Implementation Plan
- [ ] Define CLI arguments and command flow.
- [ ] Wire named patterns, file loading, and seeded generation into one coherent entrypoint.
- [ ] Implement output formatting appropriate for both tests and live terminal demos.
- [ ] Validate representative invocations manually before handing off to integration tests.
