## Goal
Provide a first-party setup flow that creates a clean evaluation repository layout with HiggsAgent, sample-project assets, and validated ticket-board wiring for new developers.

## Acceptance Criteria
- [x] `higgs-agent bootstrap sample-project <target_dir>` creates a target Git repository with copied sample-project assets and the local `.higgs/` runtime directories.
- [x] Bootstrap adds `tools/higgsagent` as a submodule, initializes nested submodules, and validates the copied sample-project ticket board through MuonTickets.
- [x] The CLI reports actionable failures for invalid sample-project names and non-empty target directories.
- [x] Regression tests cover the bootstrap success path and CLI failure paths without regressing the existing analytics CLI surface.
