## Goal
Apply structured patches and diffs from autonomous coding responses onto an existing workspace safely and observably.

## Acceptance Criteria
- [ ] Accept a structured patch or diff response shape for updating existing files after the initial scaffold exists.
- [ ] Apply edits safely against current workspace contents with conflict detection instead of blind overwrite behavior.
- [ ] Record the applied patch operations and affected paths in telemetry so later review can reconstruct what changed.
- [ ] Produce actionable failure output when a diff cannot be applied cleanly.

## Notes
- This ticket covers the second-step workflow after scaffold creation, where the agent needs to modify previously created files incrementally.
