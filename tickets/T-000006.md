## Goal
Define the execution event model, storage boundaries, and retention rules for observability and analytics.

## Acceptance Criteria
- [ ] Execution event types and required fields are defined for all core lifecycle stages.
- [ ] The boundary between committed artifacts and local-only artifacts is explicit.
- [ ] Redaction, retention, and secret-handling expectations are documented for open-source operation.
- [ ] Later analytics tickets can consume the schema without redefining base execution metrics.
