## Goal
Create directories, files, and initial project scaffolds from normalized autonomous coding-session outputs.

## Acceptance Criteria
- [ ] Materialize a normalized autonomous response into directory creation operations for nested folder structures.
- [ ] Materialize new files from structured code or text payloads without relying on manual operator copy-paste.
- [ ] Record created paths and artifact metadata so the existing write gate and observability pipeline can inspect them.
- [ ] Reject ambiguous or malformed scaffold instructions with actionable error output instead of partial silent writes.

## Notes
- This ticket covers the capability gap shown by scaffold-style model responses that describe `mkdir`, `touch`, and new file contents.
