## Goal
Run repeated autonomous ticket execution across a repository graph with checkpoint and resume support.

## Acceptance Criteria
- [x] Add a first-party command that can repeatedly execute ready tickets across a repository until a terminal condition is reached.
- [x] Persist project-run checkpoint state so interrupted runs can resume without losing ticket progress.
- [x] Ensure project orchestration reuses the Phase 6 single-ticket runtime instead of duplicating coding logic.
- [x] Surface project-level progress summaries and terminal conditions through reviewable output and telemetry.

## Notes
- This ticket turns one-ticket autonomy into a bounded repository-level loop.
