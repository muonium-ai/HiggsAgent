## Goal
Apply structured patches and diffs from autonomous coding responses onto an existing workspace safely and observably.

## Acceptance Criteria
- [x] Accept a structured patch or diff response shape for updating existing files after the initial scaffold exists.
- [x] Apply edits safely against current workspace contents with conflict detection instead of blind overwrite behavior.
- [x] Record the applied patch operations and affected paths in telemetry so later review can reconstruct what changed.
- [x] Produce actionable failure output when a diff cannot be applied cleanly.

## Notes
- This ticket covers the second-step workflow after scaffold creation, where the agent needs to modify previously created files incrementally.
- Autonomous responses now accept `patches` and `diffs` arrays containing exact-match replacement operations with repository-relative paths.
- Patch application fails when the target file is missing, the expected snippet is absent, or the snippet appears more than once, preventing ambiguous blind overwrites.
- Materialization telemetry now records patch operations alongside scaffold writes so review artifacts can reconstruct the applied edits.
