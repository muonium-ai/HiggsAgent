## Goal
Deliver the first observable dispatcher pipeline that scans ready tickets, classifies semantics, routes deterministically, executes through the selected provider, validates outputs, and records execution data.

## Acceptance Criteria
- [ ] Phase 1 scope is bounded to deterministic dispatch and excludes adaptive or benchmark behavior.
- [ ] Scanner, classifier, routing, executor, validation, tests, and docs work are represented as child tickets.
- [ ] The phase explicitly inherits the approved Phase 0 contracts.

## Notes
- Parent ticket for the MVP dispatcher phase.
- Coordination-sensitive because it defines the first real execution boundary.
