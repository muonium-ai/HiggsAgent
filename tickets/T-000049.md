## Goal
Expose a first-party configuration surface for selecting and recording the OpenRouter provider and model used for HiggsAgent evaluation runs.

## Acceptance Criteria
- [x] Guardrails configuration can declare explicit routing profiles for the economy, balanced, and deep OpenRouter routes instead of relying only on hardcoded defaults.
- [x] Route selection uses the configured provider and model identifiers, and the selected values continue to flow into the existing route and attempt-summary metadata.
- [x] Invalid routing profile configuration is rejected with actionable validation errors.
- [x] Routing tests cover profile overrides and adjacent OpenRouter and hybrid-provider tests continue to pass.
